本文档整理当前已实现的需求与结构，便于后续继续沟通与扩展。目标站点：**https://www.spelabautoparts.com/**（Shopify）。

## 三、功能与入口

### 3.1 爬取控制台

- **地址**：`http://127.0.0.1:5001`（或 `/index.html`）
- **功能**：点击「开始爬取」在本地运行爬虫；表格约 1.5 秒刷新，显示已抓取的 title、url；数据写入 `data/egr.db` 表 `pages`。
- **入口**：主页按钮「查看分类数据」「SEO 标题分析」→ 新开页 `/static/classified.html`、`/seo.html`。

### 3.2 分类数据页

- **地址**：`http://127.0.0.1:5001/static/classified.html`
- **功能**：按类型（产品页、分类页、博客、单页、首页、其他）展示数量与列表；「重新整理分类」根据当前 `pages` 表重新分类并导出。
- **依赖**：需先有抓取数据；首次使用或表空时点「重新整理分类」即可。

### 3.3 SEO 标题分析（本地 LM）

- **地址**：`http://127.0.0.1:5001/seo.html`
- **功能**：选择页面类型与条数（5/10/20/全部），点击「开始分析」后，后端**分批**调用本地 LM Studio（每批 10 条），每批完成即写入数据库并在页面**实时**展示已完成的记录；分析结束后整表覆盖为本次结果。可随时点击「查看已保存结果」从数据库加载并展示最近一次分析结果（无需重新分析）。表格列：标题、URL、评分、改进建议、3 个优化版本；标题与 URL 完整展示不截断。
- **依赖**：LM Studio 已启动并已加载模型（默认 `http://127.0.0.1:1234`）；需先有分类数据。可通过环境变量 `LM_STUDIO_BASE_URL` 修改 LM 服务地址。
- **持久化**：分析以前会清空 `seo_analysis` 表，每批完成即追加写入；打开本页或点击「查看已保存结果」时从数据库读取并展示。

### 3.4 页面类型规则（Shopify）

| 类型 | 英文标识 | URL 规则 |
|------|----------|----------|
| 首页 | homepage | 路径为 `/` |
| 产品页 | product | 路径含 `/products/` |
| 分类页 | collection | 路径含 `/collections/` |
| 博客 | blog | 路径含 `/blogs/` |
| 单页 | page | 路径含 `/pages/` |
| 其他 | other | 其余 |

规则实现：`server/classify.py` 中 `classify_page_type(url)`。

## 四、接口一览

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | / | 爬取控制台 HTML |
| GET | /static/classified.html | 分类数据页 HTML |
| GET | /seo.html | SEO 标题分析页 HTML |
| POST | /api/crawl/start | 启动爬虫（后台线程） |
| GET | /api/crawl/status | 爬取状态：running, count, error |
| GET | /api/pages | 原始页面列表（limit） |
| GET | /api/classified/counts | 各类型数量 |
| GET | /api/classified/pages | 分类列表，可选 ?page_type=product 等 |
| POST | /api/classified/rebuild | 根据 pages 重新整理分类并导出 |
| POST | /api/seo/analyze | 启动**后台分批** SEO 分析，立即返回。请求体 `{ "page_type", "limit" }`（limit 为 0 表示不限制）。返回 `{ "ok", "message", "total" }`；前端可轮询 status 与 results 获取进度与实时数据。 |
| GET | /api/seo/status | 分析进度：`{ "running", "total_expected", "total_done", "error" }`，供前端轮询。 |
| GET | /api/seo/results | 读取数据库中已保存的 SEO 分析结果（实时包含当前任务已写入的批次）。可选 `?page_type=product&limit=5000`。 |

## 五、配置与环境

### 5.1 爬虫参数（crawler/crawl_egr.py 顶部）

- `REQUEST_DELAY` / `RATE_LIMIT_INITIAL`：请求间隔（秒）
- `WORKERS`：并发线程数
- `RATE_LIMIT_MAX`：遇 429 后间隔上限
- `use_sitemap`：是否从 sitemap/robots 发现种子
- `use_shopify_api`：是否拉取 products.json/collections.json（默认关）

### 5.2 多 IP（环境变量或 .env）

- **ScraperAPI**：`SCRAPER_API_KEY=你的key`，自动走多 IP。
- **代理**：`EGR_PROXY_URL=http://user:pass@host:port` 或 `HTTP_PROXY=...`

### 5.3 端口

- 服务端口：**5001**（避免 macOS 上 5000 被 AirPlay 占用）。

### 5.4 LM Studio（SEO 分析）

- 默认 base URL：`http://127.0.0.1:1234`（OpenAI 兼容：`/v1/chat/completions`）。
- 环境变量：`LM_STUDIO_BASE_URL` 可覆盖（如 `http://192.168.1.100:1234`）。
- 需在 LM Studio 中先加载模型，分析时会自动使用当前已加载的模型。

## 六、已解决问题记录

- **只爬 1 条就结束**：起始 URL 不入队前加入 visited，避免被误判为已访问。
- **429 Too Many Requests**：全局速率限制 + 遇 429 等待 Retry-After/45s 重试 + 自动加大间隔；默认关闭 Shopify API。
- **Chrome 打不开 / 403**：服务绑定 `0.0.0.0`；若仍 403 多为本机代理/扩展，可试无痕或 Safari。
- **分类页 404**：改为通过 `/static/classified.html` 访问。
- **分类页「Unexpected token '<'」**：接口异常时保证返回 JSON；前端用 safeJson 并检测 Content-Type 为 text/html 时提示从正确地址打开并启动服务。
